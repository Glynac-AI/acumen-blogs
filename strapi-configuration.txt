Strapi Configuration Steps
Step 1: Create Content Types


# In Strapi Admin Panel
1. Go to Content-Type Builder
2. Create Collection Types:
   - Articles
   - Authors
   - Categories
   - Subcategories
   - Tags
3. Add all fields as specified above
4. Save and restart server
Step 2: Create SEO Component


1. Content-Type Builder → Components
2. Create new component category: "shared"
3. Create component: "seo"
4. Add SEO fields as specified
5. Add this component to Articles content type
Step 3: Create Category Details Component


1. Content-Type Builder → Components
2. Create component: "category-details"
3. Add "detail" field
4. Add as repeatable component to Categories
Step 4: Configure Relationships


// These are set up in the Content-Type Builder
Articles:
- author: Relation (Many-to-One) → Authors
- category: Relation (Many-to-One) → Categories
- subcategories: Relation (Many-to-Many) → Subcategories
- tags: Relation (Many-to-Many) → Tags (optional)
Authors:
- articles: Relation (One-to-Many) → Articles
Categories:
- articles: Relation (One-to-Many) → Articles
- subcategories: Relation (One-to-Many) → Subcategories
Subcategories:
- category: Relation (Many-to-One) → Categories
- articles: Relation (Many-to-Many) → Articles
Tags:
- articles: Relation (Many-to-Many) → Articles
Step 5: Set Permissions


# In Settings → Roles → Public
Allow these endpoints:
- articles: find, findOne
- authors: find, findOne
- categories: find, findOne
- subcategories: find, findOne
- tags: find, findOne
# Keep create/update/delete for authenticated users only
API Endpoints
Articles


GET  /api/articles                    # List all articles
GET  /api/articles/:id                # Get single article
GET  /api/articles?filters[slug][$eq]=your-slug  # Get by slug
GET  /api/articles?populate=*         # Get with relations
POST /api/articles                    # Create (authenticated)
PUT  /api/articles/:id                # Update (authenticated)
DELETE /api/articles/:id              # Delete (authenticated)
Authors


GET  /api/authors
GET  /api/authors/:id
GET  /api/authors?filters[slug][$eq]=author-id
GET  /api/authors?populate=articles
Categories


GET  /api/categories
GET  /api/categories/:id
GET  /api/categories?filters[slug][$eq]=category-slug
GET  /api/categories?populate[subcategories]=*
Subcategories


GET  /api/subcategories
GET  /api/subcategories/:id
GET  /api/subcategories?filters[slug][$eq]=subcategory-slug
GET  /api/subcategories?filters[category][slug][$eq]=practice-management
Tags


GET  /api/tags
GET  /api/tags/:id
GET  /api/tags?filters[slug][$eq]=tag-slug
Populate (Relations) Examples


# Get article with all related data
GET /api/articles/1?populate[author]=*&populate[category]=*&populate[subcategories]=*&populate[tags]=*&populate[featuredImage]=*&populate[seo]=*
# Get all articles with basic relations
GET /api/articles?populate[0]=author&populate[1]=category&populate[2]=subcategories&populate[3]=featuredImage
# Get author with all their articles
GET /api/authors/1?populate[articles][populate][0]=featuredImage&populate[articles][populate][1]=category
# Get category with subcategories
GET /api/categories/1?populate[subcategories]=*&populate[articles][populate][0]=featuredImage

Important Strapi Settings
1. Enable Draft & Publish


Content-Type Builder → Articles → Advanced Settings
☑ Enable draft & publish system
2. Configure Media Library


Settings → Media Library
- Set max file size (recommend 5MB for images)
- Enable responsive formats
- Set allowed formats: jpg, jpeg, png, webp
3. API Token Setup


Settings → API Tokens → Create new token
- Name: "Frontend Production"
- Token type: Read-only
- Token duration: Unlimited
- Copy token for environment variables
